Vim on Nix, Pt 1

May 2, 2018

Setting up Vim on NixOS prompted me to get more familiar with writing Nix expressions and with Vim’s initialization process.

You can use the vim_configurable package to bake plugins and configuration into Vim system-wide, rather than on a per-user basis. There’s still room for per-user configuration, but this provides a nice baseline, and means you won’t be faced with an unconfigured Vim when you need to run as root.

It’s worth mentioning sudoedit, which lets you run Vim with elevated privileges without actually switching to the root user. This means Vim will load your user’s configuration, and it’s nice for quick edits to root-owned files, but I find it less helpful when I need to edit several related files, such as the modules in my nixos configuration. Since it uses temporary files to enact the edits, you lose Vim’s persistent undo history, and if you try to navigate the filesystem relative to the file you’re editing (to open a sibling file, for example) you’ll find yourself in /var/tmp.

I’m using vim_configurable in an overlay, and I’ve named the resulting package vim-configured. I include this package in environment.systemPackages.

/nix/store

Let’s list everything in /nix/store that’s relevant to my current Vim configuration.

nix-store --query --requisites /run/current-system/sw \
  | grep -P '^[^-]+-.*?\K\bvim'

This lists 53 items, starting with a bunch of plugins,

/nix/store/0lgaf7k8sjciky1hd1n70h82mi62cznd-vimplugin-fzf-0.17.3
...
/nix/store/zkyqc2jll34xms0jbiv1zx9pbwqi88f5-vimplugin-vim-easy-align-2017-06-03

and finishing with the package that holds these plugins, a couple vimrc’s (more on these in a moment), vim_configurable, and vim (which is actually a wrapper):

/nix/store/hjhlwjk43hj6i2wqp995hjvxkghpw456-vim-pack-dir
/nix/store/a2xnidgb5hvwak5vsrb1dv29hsl1wld3-vimrc
/nix/store/63av9cp8hkwsr332nv5whpnq2k3z0iqg-nixos-vimrc
/nix/store/sz52xjgb6viyfbp7w0221hj2jr5avkgf-vim_configurable-8.0.1605
/nix/store/1jljyxpjw38gfhjc41gdla94zfimfvci-vim

So where’s the real Vim?

$ which vim
/run/current-system/sw/bin/vim
$ readlink -f $(which vim)
/nix/store/1jljyxpjw38gfhjc41gdla94zfimfvci-vim/bin/vim

So when I run vim at the command line, that’s what my PATH is pointing to. There’s another layer though:

$ cat $( readlink -f $(which vim) )
#!/bin/sh
exec /nix/store/sz52xjgb6viyfbp7w0221hj2jr5avkgf-vim_configurable-8.0.1605/bin/vim -u /nix/store/a2xnidgb5hvwak5vsrb1dv29hsl1wld3-vimrc  "$@"

It’s a wrapper script that invokes the actual executable using -u to control which vimrc gets loaded.
